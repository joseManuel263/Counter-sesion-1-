Preguntas y Respuestas

¿Qué es zone.js?

Es una librería utilizada por Angular para detectar automáticamente los cambios en la aplicación (change detection). Básicamente, hace un "parcheo" a funciones asíncronas como setTimeout, promesas, eventos del DOM, etc., para que Angular sepa cuándo actualizar la vista sin que el desarrollador tenga que hacerlo manualmente.

¿Qué es Next.js?

Es un framework de React que añade funcionalidades como renderizado del lado del servidor (SSR), generación estática de páginas (SSG), rutas automáticas, optimización de imágenes y soporte para aplicaciones híbridas. Está enfocado en construir aplicaciones web rápidas y escalables con React.

¿Qué diferencia hay en el uso de Next.js y zone.js?

- zone.js es una librería que Angular usa internamente para el manejo de detección de cambios en el frontend. - Next.js es un framework completo sobre React para desarrollar aplicaciones con renderizado en servidor, SEO optimizado y despliegue sencillo. En resumen: zone.js es un mecanismo de Angular; Next.js es un framework de React.

¿Qué es zoneless Angular?

Es una nueva forma de trabajar en Angular (a partir de la versión 17+) donde ya no se depende de zone.js para la detección de cambios. En su lugar, se usan mecanismos más explícitos como signals y effect() para actualizar la interfaz. Esto mejora el rendimiento, elimina sobrecarga innecesaria y da más control al desarrollador.
